IPP Meeting Dashboard — Guide

Purpose of this dashboard

The IPP Meeting Dashboard is a leadership report. It speaks a different language from operational monitoring reports and is meant for communicating with leadership in a broader company perspective.

How it differs from other reports

On other reports, the planned due date is the approved date from management and is used as a benchmark to see whether work has been delayed against the time of worklogs. Those are operational monitoring reports.

Here, the date you see as “Actual Production Date” is treated as a Tentative Production Date. Whether we will reach it or have already reached it is a separate discussion. This Tentative Production Date is different from the Planned Due Date of the Epic.

Delivery Status

Each epic has a Delivery Status that you set in Epics Management: Late, On-track, or Yet to start. The dashboard shows this status with bar colors and icons:

  • Late — Red bar, warning icon
  • On-track — Green bar, check mark icon
  • Yet to start — Blue bar, hour-gauge icon

This status is manually set so you can communicate clearly to leadership. The system can suggest a status based on dates, but you can always override it.

Why manual status matters

Tentative Production Date and Planned Due Date can differ in rare cases and the epic can still be on track. For example: all phases of the epic are done, but an external factor (client, contract, holidays, approvals) delays the final production date. You do not want the dashboard to auto-mark the epic as late and draw leadership attention as if the team were at fault. By setting On-track manually, you communicate that delivery is on track even though the tentative production date may differ.

Example: Epic planned due date is 15 Mar. Tentative production date is 20 Mar because of a client sign-off delay. You set Delivery Status to On-track so leadership sees the right message.

Hierarchy on this dashboard

RMIs are shown in the order: Project → Input → Component → Epic. When an epic has no Input (Product Categorization) or no Component set in Epics Planner, those levels are omitted in the tree so the epic still appears directly under the level that is set (e.g. under Project only, or under Project → Input).

Where to set Delivery Status

Open Epics Planner (Epics Management), find the epic, and use the Delivery Status column. You can change it anytime; the IPP Meeting Dashboard will reflect your choice in bar colors and icons.
